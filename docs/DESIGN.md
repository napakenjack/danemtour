# Дизайн-система сайта

Справка для будущих чатов/разработчиков: что и почему так сделано, чтобы не ломать систему
случайным несовпадающим значением.

## Референс

Редизайн (2026-07-21) ориентировался на скриншот лендинга в стиле «Triplio» (тревел-сайт,
тёмный hero на фото гор, крупный шрифтовой заголовок, карусели карточек, светлая секция статистики,
тёмный футер). Копировали **композицию и язык интерфейса**, не контент — тексты и данные остались
про danem tour (Алматы, реальные направления, реальные фичи бизнеса).

## Токены — `src/index.css` (`@theme`)

Все цвета/тени заданы как CSS-переменные Tailwind v4 (`@theme`). Не хардкодить hex в компонентах —
всегда через классы `bg-ink-*`, `text-brand-*` и т.д.

- **`brand`** (фиолетовый) — фирменный цвет клиента. Используется как акцент: ссылки, иконки,
  hover-состояния, крупная цифра в статистике. Не основной цвет кнопок.
- **`ink`** — основная нейтральная шкала. Светлый конец (`ink-50/100`) — тёплая «бумага» для фона
  страницы; тёмный конец (`ink-900/950`) — глубокий тёмно-синий, используется для hero-оверлея,
  футера, основного текста кнопок.
- **`sunset`** (оранжевый) — акцент для горящих туров/акций и предупреждений в формах. Не путать
  с brand — это отдельный смысловой цвет.
- **`--shadow-soft` / `--shadow-card` / `--shadow-float`** — три уровня теней вместо
  «на глаз». `float` — для плавающих карточек/бейджей поверх фото.
- **`--container-content` (1280px)** и **`--container-wide` (1440px)** — используются через
  утилиты `.content-container` / `.content-container-wide`. Wide — там, где нужна карусель
  или широкая композиция (Header, секции туров/отзывов на главной).

Утилиты в `@layer utilities`:
- `.eyebrow` — маленькая подпись над заголовком секции (uppercase, tracking).
- `.display-hero` — стиль огромного заголовка hero (font-display, extrabold, uppercase, tight tracking).
- `.no-scrollbar` + `.snap-x-mandatory` / `.snap-center-item` — горизонтальные карусели без
  видимого скроллбара со scroll-snap (карточки туров, отзывы).

## Типографика

- Display (заголовки) — `Golos Text` (поддержка кириллицы). Sans (текст) — `Inter`.
- Заголовки секций: `text-[2rem] sm:text-[2.5rem]`, `leading-[1.08]`, `tracking-tight`.
- Hero: `.display-hero` c `clamp`-подобной ручной шкалой `text-[3.1rem] sm:text-[5.2rem] lg:text-[6.6rem]`.

## Компоненты (`src/shared/ui`)

- **`Button`** — варианты `primary` (тёмная пилюля, дефолт), `light` (белая, для тёмных секций/hero),
  `outline`, `outlineLight` (для текста на фото), `ghost`, `sunset`, `whatsapp` (фирменный зелёный,
  не трогать цвет — это WhatsApp brand color).
- **`TourCard`** — портретная карточка (`aspect-[4/5]`), фото на всю карточку, градиент снизу,
  цена/рейтинг поверх фото. Используется и в карусели на главной, и в каталоге `/tours`.
- **`SectionHeading`** — eyebrow + заголовок + описание, используется как базовый заголовок секции
  там, где не нужна кастомная компоновка (сложные секции на главной собирают заголовок вручную,
  чтобы разместить второй элемент справа — фильтр, CTA, стрелки карусели).
- **`Reveal`** — обёртка motion для появления при скролле (`fade + translateY`). Используется
  почти на каждом блоке контента.

## Секции главной страницы (`src/pages/home`)

Порядок: `Hero` → `FeaturedTours` (карусель туров) → `Advantages` (сплит «О нас» + 4 фичи) →
`Stats` (счётчик + 4 карточки статистики) → `Experience` (фото/видео-витрина) → `ReviewsPreview`
(коллаж фото + карусель отзывов) → `FaqPreview` (сплит фото + аккордеон) → `CtaSection`
(финальный блок с формой заявки).

- **`Header`** — с 21.07.2026 всегда сплошной белый (`bg-white/90 backdrop-blur-md`), на всех
  страницах, без исключений. Раньше был прозрачным поверх hero на главной до скролла — убрано,
  когда фото hero перестало быть full-bleed (см. ниже): нав больше не сидит поверх фото, а поверх
  обычного фона страницы, так что прозрачность выглядела бы битой (белый текст на белом). `<main>`
  в `RootLayout.tsx` всегда с `pt-20` (компенсация высоты фиксированного хедера) — тоже без
  исключения для главной.
- **`Hero`** (`pages/home/Hero.tsx`) — фото-фон **не full-bleed**: вписанная карточка внутри
  `content-container-wide`, `rounded-[2rem]` (`2.5rem` от `sm:`), `min-h-[440/540/620px]` (mobile/
  sm/lg). Внутри карточки — `HeroBackground`/`useHeroImages` (карусель `hero-cover-N.png`,
  кроссфейд, уважает `prefers-reduced-motion`) без изменений, поверх — тот же маркетинговый текст,
  что и раньше (бейдж рейтинга, `display-hero` заголовок, абзац, WhatsApp-кнопка, список
  направлений), только компактнее по размеру шрифтов/паддингов под укороченную карточку.
- **`BookingWidget`** (`src/features/leads/ui/BookingWidget.tsx`) — виджет поиска с 4 вкладками
  (Отели/Авиабилеты/Ж-д билеты/Аренда машин), визуально «плавает» на стыке hero-карточки и
  следующей секции (`-mt` на обёртке поверх `content-container-wide`). Заменил `HeroLeadBar`
  (удалён). Поля — нативные (`input[type=date/time/number]`, `select`, `radio`/`checkbox`) в
  обёртке `FieldBox` (иконка + uppercase-лейбл + инпут на `bg-ink-50 rounded-2xl`, тот же паттерн,
  что был в `HeroLeadBar`). Это **не форма заявки в Supabase** — кнопка «Найти» на каждой вкладке
  собирает заполненные поля в текст и открывает `wa.me` с этим текстом (см. `whatsappLink` в
  `shared/config/site.ts`). Осознанная заглушка до реальной интеграции/CRM — данные никуда не
  сохраняются, кроме как в открывшемся чате WhatsApp.

## Осознанные плейсхолдеры, требующие подтверждения клиента

Цифры в `Stats.tsx` (`2500+`, `98%`, рейтинг `4.9`) — правдоподобные, но **не проверенные**
значения для целостности макета. Подставить реальные (или согласовать с клиентом формулировки)
до продакшен-деплоя. См. `docs/CHANGELOG.md`, запись редизайна.

## Чек-лист при добавлении новой секции

1. Токены/утилиты выше — не изобретать новые цвета/тени.
2. `Reveal` на верхнеуровневый блок секции.
3. `content-container` (1280) по умолчанию, `content-container-wide` (1440) — только если внутри
   карусель/широкая сетка.
4. Пустое/загрузочное/ошибочное состояние — если секция тянет данные из Supabase (см.
   `StateMessage.tsx`, `Skeleton.tsx`).
5. Проверить на 360 / 768 / 1440 — секции главной используют горизонтальный скролл на мобиле
   вместо сжатия сетки в 1 колонку (см. `FeaturedTours`, `ReviewsPreview`).
