# Фото для сайта — что и как называть

Сайт использует два типа изображений:

1. **Статичные фото интерфейса** — лежат в `public/images/`, подставлены как плейсхолдеры
   (сгенерированы кодом, с подписью прямо на картинке). Замените файл с тем же именем — код
   менять не нужно.
2. **Фото туров и блога** — берутся из Supabase (`tours.image_url`, `blog_posts.cover_image_url`),
   сейчас там ссылки на Unsplash. Их меняют через Supabase Dashboard (Table Editor), а не файлом.
   См. раздел «Фото туров» ниже.

## 1. Статичные фото — `public/images/`

Скопируйте фото в `danemtour-agency/public/images/`, назвав файл **точно так же**, как указано
в колонке «Имя файла» (регистр важен). Формат — JPG, до ~500 КБ после сжатия (используйте
[squoosh.app](https://squoosh.app) или аналог).

| Имя файла | Размер (Ш×В) | Где используется | Что снять / подобрать |
|---|---|---|---|
| `hero-cover-1.png`, `hero-cover-2.png`, `hero-cover-3.png`, ... | 1920×1280 (альбомная) | Главная страница, карусель фона первого экрана (авто-смена фото раз в 6 секунд) | Яркие кадры путешествий — горы, море, турист с чемоданом. Низ каждого кадра должен быть тёмным / не перегруженным деталями — на него ложится текст и градиент. |
| `journey-cover.jpg` | 1000×1250 (портрет) | Блок «Всё начинается с вас» | Клиент с сотрудником агентства, сборы в дорогу или атмосферное фото аэропорта/чемоданов. |
| `experience-cover.jpg` | 1600×900 (альбомная) | Блок «Впечатления, ради которых стоит лететь» | Самый красивый кадр из тура — пляж, отель, достопримечательность. На него накладывается кнопка Play (ссылка на Instagram). |
| `faq-cover.jpg` | 1000×1250 (портрет) | Блок FAQ на главной | Турист с чемоданом / в аэропорту — нейтральный, спокойный кадр. |
| `cta-cover-1.jpg` | 700×900 (портрет) | Финальный блок с формой заявки, левая картинка | Пляж / отель / бассейн. |
| `cta-cover-2.jpg` | 700×900 (портрет) | Финальный блок с формой заявки, правая картинка | Счастливые туристы / компания друзей в поездке. |
| `logo.svg` | любой, высота ~72px (2×36px под retina) | Хедер и остальные места на светлом фоне | Логотип в варианте для светлого фона (тёмный/цветной). |
| `logo-dark-bg.svg` | так же | Футер и мобильное меню (тёмный фон) | Тот же логотип в варианте для тёмного фона (светлый/белый). Если отдельного варианта нет — можно положить копию `logo.svg` под этим именем, но проверьте, что он читаем на тёмном фоне. |
| `favicon.svg` | SVG, квадрат | Вкладка браузера | Иконка сайта. Заменяет текущий плейсхолдер-иконку. |
| `favicon-32.png` | 32×32 PNG | Вкладка браузера (фолбэк для браузеров без поддержки SVG-favicon) | Не обязателен, если есть `favicon.svg`. |
| `apple-touch-icon.png` | 180×180 PNG | Иконка на экране iOS при добавлении сайта на главный экран | — |
| `og-cover.jpg` | 1200×630 (альбомная) | Превью ссылки в WhatsApp/Telegram/соцсетях при пересылке ссылки на сайт | Яркая картинка с логотипом/направлением, читаемая в маленьком превью. |

**Логотип и favicon — тоже просто файлом.** `Logo.tsx` и `index.html` уже подключают файлы выше по
именам; пока их нет в `public/images/` (или `public/` для favicon), сайт использует текущий
плейсхолдер (иконка компаса + текст, обычная favicon-иконка) — ничего не сломается. Как только вы
положите файл под тем же именем — он подхватится автоматически, код трогать не нужно.

Hero — единственное исключение из формата JPG: файлы **PNG**, названные `hero-cover-1.png`,
`hero-cover-2.png`, `hero-cover-3.png` и так далее. Нумерация должна идти подряд с 1, без
пропусков — сайт подхватывает фото автоматически и добавляет их в карусель, останавливаясь на
первом отсутствующем номере (если положить `hero-cover-1.png` и `hero-cover-3.png`, но не
`hero-cover-2.png`, третье фото не подхватится). Можно оставить одно фото (`hero-cover-1.png`) —
тогда карусель просто не будет переключаться. Если не положить ни одного пронумерованного файла,
сайт покажет `hero-cover.png` как запасной вариант.

## 2. Фото туров — через Supabase, не файлом

Каждый тур в таблице `tours` (Supabase → Table Editor → `tours`) имеет поле `image_url` — прямая
ссылка на изображение. Чтобы заменить фото тура:

1. Загрузите фото в Supabase Storage (или на любой хостинг картинок) и скопируйте публичную ссылку.
2. Table Editor → `tours` → нужная строка → вставьте ссылку в `image_url`.
3. Обновление на сайте — сразу, без деплоя.

Рекомендуемый размер фото тура — **1200×1500** (портрет, соотношение 4:5), карточки на сайте
кадрируют изображение под этот формат.

То же самое для блога — поле `cover_image_url` в таблице `blog_posts`, размер **1600×900**.

## 3. Чек-лист перед заменой

- [ ] Реальное фото не хуже по резкости/освещению плейсхолдера
- [ ] Нет чужих логотипов/лиц без разрешения на использование
- [ ] Формат JPG/WebP, оптимизировано (не больше 500 КБ на статичные, не больше 300 КБ на карточки туров)
- [ ] Портретные фото — не растянуты и не обрезают лица/важные детали по краям
